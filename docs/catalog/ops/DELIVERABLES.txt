🔍 METRICS PER SYSTEM

For each operation, displays:
  • Status (running, idle, success, error, scheduled)
  • Cycles/jobs completed
  • Success rate
  • Best accuracy/metrics
  • Resource usage (CPU/memory)
  • Current task and progress
  • Active cycles/jobs
  • Last update timestamp

🎨 OUTPUT FORMATS

1. Full View (Default)
   • Detailed metrics for each operation
   • Progress bars
   • Alert indicators
   • Organized by category

2. Compact View (--compact)
   • One line per operation
   • Status emoji + name + status
   • Minimal output

3. Watch Mode (--watch)
   • Auto-refreshes every 5 seconds
   • Perfect for development
   • Press Ctrl+C to exit

4. Problems Only (--problems)
   • Shows only items with alerts/errors
   • High CPU, failed jobs, etc.
   • Useful for alert-focused monitoring

5. JSON Export (--export)
   • Machine-readable format
   • Perfect for CI/CD integration
   • Writes to: data_out/auto_ops_dashboard.json

🚨 ALERT SYSTEM

Detects and displays:
  • Critical alerts (🔴):
    - CPU usage >80%
    - Memory usage >80%
    - Disk usage >85%
  • Warnings (⚠️):
    - Promotion failures
    - Accuracy decline
  • Errors (❌):
    - Failed jobs
    - Validation failures
  • Degradation (📉):
    - Performance trends
